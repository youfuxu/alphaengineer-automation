# Threads Hook 測試 — Day 11 判讀框架

判讀日：2026-07-09（測試跑滿 6/29–7/08 共 20 支後隔天）
資料來源：`posts/hook-test.json`（20 支純文字 Hook）
發文邏輯：每天 2 支（AM 台灣 20:30 / PM 台灣 22:00），10 天發完。

> 核心原則：**只看留言（comments）+ 轉發（reposts / quotes），不看讚。**
> 讚是被動、低成本的訊號，容易被演算法灌水；留言與轉發才代表「觀眾願意花力氣互動 / 幫你擴散」，是真正能驗證 Hook 強度的行為訊號。

## 1. Hook 類型對照

測試把 20 支分成 4 種 Hook 開場型，各 5 支：

| Hook 類型 | 開場策略 | 對應 index |
|---|---|---|
| `num` | 具體數字開場（$174、$0、6 小時→22 分…） | 0, 4, 8, 12, 16 |
| `process` | 過程/幕後坦白（「我自動化 100 篇後真正壞掉的是…」） | 1, 5, 9, 13, 17 |
| `identity` | 身份認同/挑釁（「工程師是科技業最被低估的聰明人」） | 2, 6, 10, 14, 18 |
| `fear` | 恐懼/急迫（「你的工程師工作大概剩 5 年」） | 3, 7, 11, 15, 19 |

判讀時除了看單支表現，也要**聚合看哪一種 Hook 類型平均最強**。

## 2. 互動分計算表（模板）

**互動分公式：**

```
互動分 = (留言數 × 3) + (轉發數 × 5)
```

- 權重理由：轉發 = 觀眾願意把你的內容掛在自己名下擴散，價值高於留言，故給 5；留言代表投入但不擴散，給 3。

## 3. 前 3 名共同點分析

依互動率排序，取前 3 名，逐欄拆解找共同結構：

| 分析欄位 | 第 1 名 | 第 2 名 | 第 3 名 |
|---|---|---|---|
| index / ref |   |   |   |
| Hook 類型（num/process/identity/fear） |   |   |   |
| 第一句字數（越短越好？） |   |   |   |
| 開場是否含具體數字 |   |   |   |
| 情緒基調（憤怒/恐懼/認同/好奇/自嘲） |   |   |   |
| 是否點名受眾（「工程師」「你」） |   |   |   |
| 結尾提問型態（是非題/選邊站/自我揭露邀請） |   |   |   |
| 貼文行數 / 是否用短句斷行 |   |   |   |
| slot（AM/PM）|   |   |   |
| 是否有「反直覺 / 打破迷思」轉折 |   |   |   |

**要回答的三個問題：**
1. 前 3 名是否集中在同一種 Hook 類型？（若是 → 那類型就是主力）
2. 前 3 名的**結尾提問**有沒有共同型態？（Threads 互動主要靠留言，結尾問法往往比開場更關鍵）
3. 有沒有跟 slot（發文時段）相關？AM 還 PM 表現好？

## 4. 爆款公式推導步驟

1. **鎖定贏家 Hook 類型**：從第 2 節聚合表挑平均互動率最高的 1–2 種類型。
2. **拆解開場句式**：把前 3 名的第一句抽象成模板，例如
   `num` → 「我花了 $X 在 [某事]，它做了 $Y 的工作。」
   `fear` → 「你的 [身份] 大概剩 X 年。不是因為 A，而是因為 B。」
3. **固定結尾提問公式**：把前 3 名結尾問句抽象成 2–3 個可重複套用的提問模板（選邊站型 / 自我揭露型 / 是非題型）。
4. **定調情緒 + 受眾點名**：確認贏家共同的情緒基調與是否直接點名「工程師 / 你」，寫進公式。
5. **輸出一句話公式**：例如
   `[反直覺數字或斷言開場] + [1 句展開理由 with 具體對比] + [選邊站式結尾提問]`
6. **驗證**：用公式回頭套 3 支本次「非前 3 名」的主題，檢查是否也會變強（sanity check，不是真的發）。
